# Release

Hermes: Godspeed is distributed as a source repository plus a generated static web build.

## Release Model

- The npm package is intentionally marked `"private": true`; npm publication is not the release target.
- `dist/` is generated by `npm run build` and is ignored in source control.
- Public hosting should deploy the contents of `dist/` to a static web host.
- Test-only hooks are enabled only in Vite development or the dedicated Playwright test build on local hosts such as `localhost` and `127.0.0.1`.

## Release Check

Run this before publishing:

```bash
npm ci
npx playwright install
npm run release:check
npm audit --omit=dev --audit-level=moderate
```

`npm run release:check` verifies TypeScript compilation, unit tests, Playwright e2e tests against a production-style local preview, a final production build, and a release-bundle scan that rejects shipped test/debug hooks. The e2e suite fails on browser console errors, page errors, request failures, and horizontal overflow.

`npx playwright install` is a one-time prerequisite that downloads the browser binaries the e2e suite drives; run it before `npm run test:e2e` or `npm run release:check`.

`npm audit --omit=dev` excludes dev-only tooling (for example the `voice:intro` Kokoro script) that is never included in the shipped static artifact in `dist/`. The `trailer/` directory is a separate Remotion project and is not part of the game build or the released artifact.

## Manual QA

Before a public release, verify:

- First-load intro appears, can be skipped, and transitions to the start panel.
- Desktop keyboard controls work: left, right, jump, slide, pause, resume.
- Mobile touch controls work on a 390px-wide viewport.
- Daily Dispatch starts and produces a share summary at game over.
- Recording controls disable cleanly when MP4 recording is unsupported.
- Real art assets render in the intro and gameplay; Phaser missing-texture checkerboards must not appear.
- No horizontal overflow exists on desktop or mobile.

## Asset Approval

Bundled image assets are tracked in `ASSETS.md`. Do not publish a release with replaced or newly added assets until `ASSETS.md` is updated with the release status for those files.
